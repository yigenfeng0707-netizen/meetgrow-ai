### T07 增强renderMarkdown支持表格渲染

**目的**：AI输出的Markdown表格能在产品中正确渲染。

**修改位置**：`MeetGrow_AI.html` 第1267-1296行 `renderMarkdown` 函数。

**执行方法**：在现有正则替换链中，在 `<blockquote>` 处理之后、`<ul>` 处理之前，增加表格解析：

```javascript
// Table support
.replace(/^\|(.+)\|$/gm, function(match, content) {
  const cells = content.split('|').map(c => c.trim());
  if (cells.every(c => /^[-:]+$/.test(c))) return '<!--table-sep-->';
  return '<tr>' + cells.map(c => '<td>' + c + '</td>').join('') + '</tr>';
})
```

然后在函数末尾增加 `<tr>` 包裹处理：

```javascript
html = html.replace(/((?:<tr>.*?<\/tr>\s*(?:<br>)?)+)/g, function(match) {
  return '<table>' + match.replace(/<!--table-sep-->/g, '').replace(/<br>/g, '') + '</table>';
});
```

**验收标准**：含Markdown表格的方案输出能正确显示为HTML表格，带边框样式。

### T08 Word导出改为.docx扩展名

**目的**：提升导出文件的专业感。

**修改位置**：`MeetGrow_AI.html` 第1356行。

**具体修改**：
```javascript
// 原来

// 改为
downloadFile(html, `${plan.name}-增长方案.docx`, 'application/vnd.openxmlformats-officedocument.wordprocessingml.document');
```

**注意**：实际内容仍为HTML格式，但.docx扩展名在现代Word中能正确打开。如需真正的.docx需引入JSZip库，当前方案是性价比最高的折中。

**验收标准**：导出的文件扩展名为.docx，Word/WPS能正常打开。
